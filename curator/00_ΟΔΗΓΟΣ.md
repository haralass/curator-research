### Τι είναι το Curator
macOS desktop app για **αυτόματη οργάνωση αρχείων** — κυρίως ~/Downloads. Χτίζεται από έναν CS φοιτητή στην Κύπρο. Τρέχει **100% locally**, χωρίς cloud, χωρίς subscriptions.

### Stack
| Layer | Τεχνολογία |
|---|---|
| Frontend | Tauri 2 + React 19 + TypeScript |
| Backend | Rust |
| AI / Python | ~/Library/Scripts/ai_classifier.py — τρέχει με /opt/anaconda3/bin/python3 |
| LLM inference | Ollama (local) |
| Embeddings DB | ChromaDB |
| Python έχει ήδη | hdbscan, umap-learn, chromadb, ollama, markitdown, unisim, fastembed |

### Βασικές Αρχιτεκτονικές Αποφάσεις (έχουν παρθεί, δεν επαναδιαπραγματεύονται)
1. **Dynamic discovery** — μηδέν predefined categories. Τα clusters βγαίνουν από τα ίδια τα αρχεία.
2. **Embedding model** → BGE-M3 (πολύγλωσσο, Greek-aware, 8192 ctx, official Ollama) — αντικαθιστά το nomic-embed-text που είναι English-only.
3. **Pipeline**: BGE-M3 embed → UMAP (10-dim) → HDBSCAN → HERCULES naming → DenStream (streaming updates)
4. **UMAP είναι ΥΠΟΧΡΕΩΤΙΚΟ** πριν HDBSCAN (το HDBSCAN spec λέει max ~50-100 dims, έχουμε 1024).
5. **DenStream** → χρησιμοποιούμε issamemari/DenStream ή waylongo/denstream, ΟΧΙ River (έχει 7 confirmed bugs).
6. **GLOSH scores** (από HDBSCAN) → τροφοδοτούν απευθείας το Antilibrary feature.
7. **Folder depth**: 3 επίπεδα — HERCULES δημιουργεί sub-categories αναδρομικά.
8. **Approval UX**: 3-tier confidence (≥80% pre-checked, 50-79% unchecked, <50% individual attention), max 50-100 items/session.
9. **Deep Reorganize**: re-runnable, δείχνει μόνο diff από την τελευταία φορά.
10. **Antilibrary**: useCount==0 αρχεία → πρόταση διαγραφής (GLOSH >0.7 = genuine outlier).
11. **Versions**: RETSim >0.86 → auto _versions/, 0.72-0.86 → Review.
12. **Unmatched νέα αρχεία** → Review folder.
