## ΜΕΡΟΣ 7: ΑΝΑΛΥΤΙΚΗ ΑΞΙΟΛΟΓΗΣΗ

### Τι είναι σίγουρα σωστό

1. **DenStream για streaming** — validated από paper 2026 (Silhouette 0.685 vs DBSTREAM 0.327)
2. **RETSim για versions** — Google ICLR 2024, production-tested
3. **HDBSCAN για batch** — δεκάδες papers, production-proven
4. **k-LLMmeans naming** — paper 2025, streaming-capable
5. **macOS metadata signals** — deterministic, zero latency, zero ML
6. **Approval workflow** — Terraform/Ansible pattern, battle-tested σε infrastructure
7. **Potential siblings UI** — Dropbox Smart Move, validated σε production

### Τι έχει ρίσκο

1. **Project detection με NER** — novel, άρα αναπόδεικτο για personal files. Μπορεί να δώσει false positives αν δύο εντελώς άσχετα αρχεία αναφέρουν τυχαία το ίδιο entity.

2. **Sub-folder hierarchy** — αγγλικά: "the more levels, the less the user uses the hierarchy." Ο Finder γίνεται δύσχρηστος με >3 επίπεδα. Προτείνω max 2 επίπεδα.

3. **DenStream tuning** — το `epsilon` parameter πρέπει να calibraριστεί στα δικά σου embeddings. Λάθος τιμή → υπερβολικά πολλά ή λίγα clusters. Χρειάζεται k-NN distance plot πρώτα.

4. **Performance** — 5000 αρχεία × content extraction. Εκτίμηση: ~45 λεπτά με OLLAMA_NUM_PARALLEL=4. Αν το Ollama κολλήσει, το SQLite queue επιτρέπει resume.

5. **Temporal co-occurrence** — FSEvents παρακολουθεί writes, όχι reads. Άρα η "co-access" είναι co-modification, όχι co-open. Μικρότερο signal.

### Τι είναι genuinely novel και κανένας δεν έχει κάνει

1. Antilibrary με `kMDItemUseCount` για file organizer
2. File biography (value score) ως organization signal
3. NER co-occurrence για cross-format project detection
4. Semantic gravity correction (`log(cluster_size)` penalty)
5. Graph betweenness centrality για approval UI prioritization
6. Folder biography (completeness comparison across semesters)

### Σύσταση Υλοποίησης (σειρά προτεραιότητας)

**Sprint 1 — Foundation (ένα Σαββατοκύριακο)**
- SQLite schema για file catalog
- Scan + cache (mtime+size)
- nomic-embed-text batching
- Basic HDBSCAN clustering
- JSON manifest output

**Sprint 2 — Duplicate Detection (ένα Σαββατοκύριακο)**
- SHA-256 exact duplicates
- imagededup για images
- RETSim (unisim) για versions
- MinHash για near-duplicate text

**Sprint 3 — Smart Features (δύο Σαββατοκύριακα)**
- DenStream για ongoing routing
- k-LLMmeans για folder naming
- Antilibrary (kMDItemUseCount)
- macOS metadata pre-signals

**Sprint 4 — Advanced (optional)**
- NER project detection
- Graph betweenness approval UI
- Sub-folder hierarchy
- Folder biography

### Τι κρατάμε από το παλιό σύστημα

- Rust backend (σχεδόν όλο αμετάβλητο)
- UI pages (μικρές αλλαγές)
- Fast path extensions (.mp3, .zip, .dmg) — πάντα σωστά, χωρίς AI
- ChromaDB (ήδη έχουμε embeddings)
- Undo log pattern
