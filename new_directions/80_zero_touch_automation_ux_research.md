# Zero-Touch Automation UX: What the Literature Says About Appropriate Automation in PIM
**File:** 80 | **Date:** 2026-06-01 | **Status:** Research Complete

## Summary
File 77 established that aggressive silent automation impairs users' mental models of their own files (photo-taking impairment analog). This file consolidates the broader literature: "automation surprise" in PIM is a documented failure mode; mixed-initiative systems where AI proposes and humans decide are consistently preferred; automation bias (over-trusting AI suggestions) is real but reducible via friction and explanation. The Parasuraman-Sheridan framework places "appropriate" personal file organization automation at Level 4–5 out of 10 — the system narrows options and suggests, but the human always executes. Zero-touch silent auto-move sits at Level 8–9 and is contraindicated by the literature.

## Key Findings

### Automation Surprise in PIM

- **iCloud Drive / Optimize Mac Storage**: The most documented real-world automation surprise in macOS. When "Optimize Mac Storage" is enabled, macOS silently evicts files to iCloud when local disk space is low. Users report: (1) files disappear from Spotlight search (iCloud-evicted files cannot be indexed locally), (2) users cannot predict which files are local vs. cloud-only, (3) the system provides no pre-eviction warning. Apple Community discussions confirm this is a widespread complaint with no user-facing remedy.
- **Time Machine**: Users frequently report discovering that Time Machine excluded categories of files without notification — "I thought it was backing up everything." The exclusion list is set silently by the OS.
- **Spotlight index corruption**: Batch file renames or moves by automation tools cause Spotlight to lose track of files, producing the experience of "the file exists but I can't find it."
- Common thread: **silent automation that moves or reclassifies files without notifying the user destroys the user's internal map of their filesystem**.

### The "Folder Amnesia" Failure Mode

No published research uses the exact term "folder amnesia," but the phenomenon is well-documented under related names:
- **"Navigation disorientation"** in PIM literature: users lose the spatial-cognitive map of where files live when the folder structure changes underneath them.
- **Scientific Reports (2015)**: "Navigating through digital folders uses the same brain structures as real world navigation" — hippocampal place-cell analogs activate during folder navigation. This explains why silent reorganization is not a minor UX annoyance but a disruption of genuine spatial memory encoding.
- **PIM anxiety research**: Key affective challenge identified is "anxiety about information loss" — users experience distress specifically when they cannot verify where files have gone. Automated moves without receipts trigger this.
- **"What's in People's Digital File Collections?" (2024, arXiv:2402.06421)**: Empirical study of 348 file collections finds that personal collections have far more images than expected, with idiosyncratic organizational patterns that reflect individual mental models. No two users organize the same way — which means any automated reorganization that imposes a canonical taxonomy directly conflicts with pre-existing mental models.

### Mixed-Initiative Systems: The Evidence Base

- **Definition**: Mixed-initiative systems share initiative between human and AI — the AI proposes, the human confirms, either party can lead.
- **AI UX Design Patterns (2024)**: Mixed-Initiative Control is an established design pattern with specific requirements:
  - Human input always takes precedence
  - Agent should never block human from interacting
  - Clear indicators of who currently has initiative
  - Interrupt-without-disruption capability
- **File organization specifically**: "Suggestions proposed by the AI take effect only upon user approval, making the process one of continuous negotiation." This framing (from Contextify, arxiv:2604.07121) captures the right mental model.
- **Human-AI Collaboration (Holter 2024, CGF)**: Split-screen / shared-workspace collaborative modalities "balance AI autonomy with user oversight" and are "valued for enabling flexible division of labor." Full autonomy was less trusted and produced worse outcomes.

### Automation Bias: Evidence and Magnitude

- **Springer AI & Society (2025)**: Comprehensive review confirms automation bias is robust: "the mere knowledge of advice being generated by an AI causes people to overrely on it, even when it contradicts available information."
- **ScienceDirect 2024**: Experimental study found users follow AI advice even when contradictory cues are visible. "AI literacy" (self-reported) did not reduce automation bias — knowing about AI limitations does not prevent over-reliance.
- **Key finding for Curator**: Users presented with a single AI-generated folder suggestion will tend to accept it without critical evaluation, even when it is wrong. This is an argument *for* showing prediction sets (multiple candidates) rather than a single forced suggestion — it breaks the "AI said so" heuristic by forcing the user to choose.
- **Mitigation evidence**: Warning nudges embedded in UI "almost doubled user performance" over purely faulty AI support conditions. Counter-explanations (why the AI might be wrong) reduce inappropriate reliance by 38%.

### Parasuraman-Sheridan Levels of Automation Applied to File Organization

The Parasuraman, Sheridan & Wickens (2000) framework defines 10 levels of automation from "human does everything" (L1) to "AI acts without informing human" (L10).

Applied to file organization actions:

| Level | Description | File Organization Analog |
|---|---|---|
| L1 | Human does everything | User manually drags files to folders |
| L2 | Computer offers complete alternatives | AI shows full suggested reorganization plan |
| L3 | AI narrows to a few alternatives | AI suggests 2–3 candidate folders per file |
| L4 | AI suggests one option | AI recommends one folder, human confirms |
| L5 | AI executes if human approves | Approve → move; reject → stays |
| L6 | AI executes, human has brief veto window | Move happens after 10s delay, undo available |
| L7 | AI executes, auto-informs human | Move happens, notification shown |
| L8 | AI executes, informs only if asked | Move happens silently, retrievable from log |
| L9 | AI executes, tells human only if it decides to | Fully silent auto-move, occasional summaries |
| L10 | AI executes, never informs | Black-box automatic reorganization |

**Literature recommendation**: For personal file management, L4–L5 is the appropriate range. L6 (brief veto window) is acceptable. L7 and above cross into automation surprise territory for personally meaningful content.

### Design Recommendations: Friction Level

Research converges on a specific set of design requirements:
1. **Never silently move a file.** Even L7 (notification after the fact) is risky. L5 (explicit approval before move) is the safety floor.
2. **Show the reason, not just the suggestion.** "Move to Work/CoolProject because 3 related files are already there" outperforms "Move to Work/CoolProject." Counter-explanation reduces automation bias.
3. **Prediction sets over point predictions.** When confidence is low, show 2–3 options. Forces user engagement with the decision. Also produces better calibration alignment (see File 79).
4. **Undo is not sufficient as the sole safety net.** Users who don't know a move happened cannot undo it. The undo must be paired with a visible record of all automated actions.
5. **Approval friction should scale with file importance.** Low-friction batch approval (swipe to accept all) for screenshots/temp files; high-friction single-file approval for documents with project entities in their content.
6. **Periodic "audit mode."** Monthly summary: "Curator organized X files into Y folders. Here's what changed." This rebuilds mental model integrity without requiring per-file engagement.

### Measuring Whether Users Can Still Navigate Their Filesystem

No established benchmark exists for "filesystem navigation ability after AI reorganization" — this is an open research gap. Proposed metrics based on adjacent literature:

1. **Time-to-file retrieval**: Present user with a file they created 3 months ago; measure time to locate it. Baseline before AI; compare after 3 months of AI-assisted organization.
2. **Mental model accuracy test**: Ask user to describe where specific files are (folder path). Score against ground truth. Run at month 0 and month 3.
3. **Unaided navigation rate**: % of file-finding tasks completed without using search (Spotlight). High unaided navigation = intact spatial mental model. Low rate = over-reliance on search to compensate for lost spatial memory.
4. **Recall of recent moves**: "Where did Curator move your tax document last month?" Correct answer = mental model preserved.
5. **Anxiety scale**: Adapted from PIM literature's "information loss anxiety" scale. Validate at 1-month intervals.

## Applicability to Curator

1. **The "zero-touch" marketing framing is misleading and dangerous.** Position Curator as "low-friction approval" not "fully automatic." Even one-tap approval is infinitely better than silent moves for mental model preservation.
2. **Default automation level: L5** (AI executes if human approves, proposal shown in notification/tray). Never ship L7+ as a default.
3. **Optionally expose L6** (5-second undo window after auto-move) as a power-user setting with explicit warning: "Files moved without your review. You may not remember where they went."
4. **Surface the reason**: every suggestion shown in the approval UI must include a one-line rationale ("Same project entities as X, Y, Z").
5. **Implement the monthly audit view**: a plain list of "Curator moved N files last month" with folder-level grouping. This is the anti-amnesia feature.
6. **Use prediction sets (File 79) in the approval UI**: when set size > 1, show the user the alternatives — this is both better UX (honest about uncertainty) and a proven automation-bias reducer.
7. **Importance-tiered friction**: screenshots → auto-batch-approve OK; any file with project-entity NER hits → always require explicit single-file confirmation.
8. **Measure navigation retention** in the user study (File 61/IRB): include a time-to-file retrieval test at 0, 1, and 3 months. This is the primary outcome metric for the automation UX research question.

## Open Questions

- What is the **minimum friction** that preserves mental model without causing "approval fatigue" (File 48)? The tension between Files 48 and 77/80 is unresolved empirically.
- Does **explanatory rationale** (showing why Curator suggests a move) reduce automation bias for file organization specifically, or is the effect domain-specific?
- Does the **prediction set UI** (showing 2–3 candidate folders) cause more confusion than a single confident suggestion for low-tech users?
- Is a **"trust budget"** approach viable: Curator earns silent-move rights for specific file types/folders based on demonstrated accuracy over time?
